## 1. 系統功能概覽

本系統為 LINE 聊天機器人，整合以下功能：

| 功能 | 說明 |
|------|------|
| AI 智慧行程規劃 | 用戶輸入需求，AI 自動產生澎湖行程 |
| 航班查詢 | 即時爬取華信、立榮航空班次與票價 |
| 房間預定 | 互動式填寫入住日期、人數、備註，加入心願清單 |
| 交通查詢 | 船班、租車、機車、自行車資訊，可加入清單 |
| 收藏與詢價 | 用戶整理心願清單後送出，系統自動寄 Email 通知負責人 |
| 真人客服 | 用戶輸入「真人」轉接，負責人可在 LINE OA Manager 後台回覆 |

## 2. 建立 LINE 官方帳號

### 2-1. 申請 LINE 官方帳號

1. 前往 **LINE for Business**：https://www.linebiz.com/tw/
2. 點選右上角「**立即開始**」→「建立 LINE 官方帳號」
3. 登入個人 LINE 帳號（建議使用公司專屬帳號）
4. 填寫帳號名稱（例如：澎湖旅遊）、業種（觀光旅遊）、公司資訊
5. 完成申請後，進入「**LINE Official Account Manager**」後台

### 2-2. 開啟 Messaging API

1. 在 LINE OA Manager 後台，左側選單點「**設定**」→「**Messaging API**」
2. 點「**啟用 Messaging API**」
3. 選擇或建立一個 **LINE Developers** 頻道（Channel）
4. 記錄下以下三個資訊（後續步驟需要）：
   - **Channel ID**
   - **Channel Secret**
   - **Channel Access Token**（點「發行」產生）

### 2-3. 設定 Webhook

1. 在 LINE Developers Console，進入你的 Channel
2. 找到「**Webhook URL**」欄位
3. 填入你的 Render 伺服器網址（格式如下）：
   ```
   https://你的服務名稱.onrender.com/webhook
   ```
4. 開啟「**Use webhook**」開關
5. 點「**Verify**」驗證，出現 Success 即成功

### 2-4. 關閉自動回覆（必要）

1. LINE OA Manager → 「**自動回應訊息**」
2. 將「自動回應」**關閉**（否則會與 Bot 回覆衝突）
3. 「加入好友的歡迎訊息」可保留，但內容請改為純文字說明

### 2-5. 建立 LIFF（航班查詢用）

1. 進入 LINE Developers Console → 你的 Channel → 「**LIFF**」分頁
2. 點「**Add**」新增一個 LIFF App
3. 設定如下：

   | 欄位 | 填入值 |
   |------|--------|
   | LIFF app name | 航班查詢 |
   | Size | Full |
   | Endpoint URL | `https://你的服務名稱.onrender.com/liff/flight` |
   | Scope | openid, profile |

4. 儲存後，複製產生的 **LIFF ID**（格式如 `1234567890-xxxxxxxx`）

## 3. 申請各項 API 金鑰

系統需要以下 API，請逐一申請：

### 3-1. Mistral AI（AI 行程規劃）

1. 前往：https://console.mistral.ai/
2. 註冊帳號 → 進入「**API Keys**」→ 點「Create new key」
3. 複製 API Key（格式開頭為 `sk-...` 或 `mis...`）

> 免費方案每分鐘有請求次數限制，多人同時使用時可能較慢。

### 3-2. 環境變數設定（.env 檔）

將所有金鑰整理到專案根目錄的 `.env` 檔案，格式如下：

```
# LINE Bot
LINE_CHANNEL_SECRET=你的Channel_Secret
LINE_CHANNEL_ACCESS_TOKEN=你的Channel_Access_Token

# LIFF
LIFF_ID=你的LIFF_ID

# AI（Mistral）
MISTRAL_API_KEY=你的Mistral_API_Key

# Email 通知
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=你的Gmail帳號@gmail.com
SMTP_PASS=你的應用程式密碼
NOTIFY_EMAIL=負責人收件信箱@gmail.com
```

> **注意：** `.env` 檔案絕對不可上傳到 GitHub，系統已設定忽略此檔案。

## 4. 郵件帳號設定（Gmail SMTP）

系統使用 Gmail 寄送詢價通知，需要開啟「應用程式密碼」功能。

### 4-1. 開啟兩步驟驗證

1. 前往 Google 帳戶：https://myaccount.google.com/
2. 左側點「**安全性**」
3. 找到「**兩步驟驗證**」，點「開始使用」並完成設定

### 4-2. 產生應用程式密碼

1. 在 Google 帳戶安全性頁面
2. 搜尋「**應用程式密碼**」並進入
3. 選擇「郵件」→「其他（自訂名稱）」→ 輸入「LINE Bot」
4. 點「產生」，複製 **16 位數密碼**（格式如：`xxxx xxxx xxxx xxxx`）
5. 將此密碼（去除空格）填入 `.env` 的 `SMTP_PASS` 欄位

> 使用應用程式密碼而非 Gmail 登入密碼，更安全且不會因為密碼更改而失效。

## 5. 正式上線部署（Render）

Render 是雲端代管平台，可讓系統 24 小時運行，不需要自己的電腦保持開機。

### 5-1. 準備工作

確認以下檔案已存在於專案中（開發者應已準備好）：
- `requirements.txt`（套件清單）
- `runtime.txt`（Python 版本，內容：`python-3.11.9`）
- `render.yaml`（Render 部署設定）

### 5-2. 部署步驟

1. 前往：https://render.com/ 並註冊帳號（可用 GitHub 登入）
2. 點「**New +**」→「**Web Service**」
3. 選擇「**Connect a repository**」，連接 GitHub 帳號並選擇此專案
4. 設定如下：

   | 欄位 | 填入值 |
   |------|--------|
   | Name | penghu-linebot（自訂） |
   | Region | Singapore（亞洲最近） |
   | Branch | main |
   | Runtime | Python 3 |
   | Build Command | `pip install -r requirements.txt` |
   | Start Command | `gunicorn app:app` |

5. 點「**Advanced**」→「**Add Environment Variable**」
   逐一將 `.env` 的所有變數貼入（Key / Value 分開填）

6. 點「**Create Web Service**」，等待約 3–5 分鐘完成部署

7. 部署成功後，Render 會提供一個網址（如 `https://penghu-linebot.onrender.com`）
   → 將此網址設定回 LINE Webhook URL（加上 `/webhook`）

### 5-3. 免費方案限制

Render 免費方案的 Web Service 在 **閒置 15 分鐘**後會自動休眠，首次有人傳訊息時需等待約 **30–60 秒**喚醒。

**解決方式（保持不休眠）：** 參考第 6 節「系統長期運行設定」。

## 6. 系統長期運行設定

### 6-1. 使用 Render Cron Job（定時喚醒）

1. 在 Render 後台點「**New +**」→「**Cron Job**」
2. 設定如下：

   | 欄位 | 填入值 |
   |------|--------|
   | Name | keep-alive |
   | Schedule | `*/14 * * * *`（每 14 分鐘執行一次）|
   | Command | `curl https://你的服務名稱.onrender.com/` |

3. 儲存後，系統每 14 分鐘自動 ping 一次，防止休眠

> **升級付費方案（$7 USD/月）可避免休眠問題**，適合正式營運時使用。

### 6-2. 使用 UptimeRobot（免費外部監控）

1. 前往：https://uptimerobot.com/ 並免費註冊
2. 新增「**HTTP(s)**」監控
3. URL 填入：`https://你的服務名稱.onrender.com/`
4. 監控間隔設為「**5 分鐘**」
5. 開啟 Email 通知（網站停止時自動發信告知）

## 7. 日常操作說明

### 7-1. 設定行程資料

熱門行程資料存放在專案的以下路徑：

```
data/
  popular_trips.json    ← 熱門行程內容
  knowledge_base/       ← AI 行程規劃參考資料（.txt 文字檔）
```

**修改熱門行程：**
用記事本或 VS Code 開啟 `data/popular_trips.json`，依照現有格式增減行程內容後儲存，再重新部署（推送到 GitHub）即生效。

**新增 AI 知識庫資料：**
將新的介紹文字（景點、美食、交通等）存成 `.txt` 檔放入 `data/knowledge_base/` 資料夾，再重新部署即生效。詳細說明見第 8 節。

### 7-2. 每天查看真人回覆（LINE OA Manager）

當用戶輸入「真人」後，Bot 會停止自動回覆，等待人工回覆。

**操作步驟：**
1. 開啟手機 LINE → 搜尋並開啟「**LINE Official Account**」App
   （或在電腦瀏覽器前往 https://manager.line.biz/）
2. 點選你的官方帳號
3. 點「**聊天**」→ 查看所有對話
4. 有「真人」模式的用戶，可直接在此輸入回覆
5. 用戶輸入「**結束客服**」後，Bot 自動恢復

> 建議每天早上固定查看一次，確保沒有遺漏的用戶詢問。

### 7-3. 收取詢價郵件

當用戶在「收藏清單」點選「確認詢價」後，系統自動寄信到 `NOTIFY_EMAIL` 設定的信箱。

**郵件內容包含：**
- 訂單編號（格式：PH-YYYYMMDD-HHMMSS）
- 用戶的心願清單（行程、航班、住宿、交通）
- AI 行程規劃的完整內容
- 住宿預定的入住日期、退房日期、人數、備註

**建議流程：**
每天查看信箱 → 根據訂單內容聯繫客人（LINE 真人客服或電話）→ 確認後安排服務

## 8. 新增與修改資料

### 8-1. 新增熱門行程

開啟 `data/popular_trips.json`，格式說明：

```json
{
  "海島探險": [
    {
      "id": "trip_001",
      "name": "澎湖藍洞半日遊",
      "desc": "探索神秘藍洞，浮潛體驗澎湖海底世界。",
      "tags": ["浮潛", "自然景觀"],
      "duration": "半天",
      "price_note": "NT$800 起"
    }
  ]
}
```

> `id` 必須是唯一值，不可重複。建議用 `trip_001`、`trip_002` 依序命名。

### 8-2. 新增 AI 知識庫文章

1. 用記事本新增一個 `.txt` 檔（例如：`澎湖景點介紹.txt`）
2. 內容為純文字，每段之間空一行
3. 將檔案放入 `data/knowledge_base/` 資料夾
4. 推送到 GitHub 後重新部署，AI 即可參考新資料回覆用戶

**範例文章格式：**
```
澎湖西嶼燈塔

位於澎湖西嶼鄉，建於 1778 年，是台灣歷史最悠久的燈塔之一。
燈塔周圍視野開闊，可眺望菊島海峽，夕陽景色壯觀。

開放時間：週二至週日 09:00–17:00
交通方式：從馬公搭船約 30 分鐘抵達西嶼，再騎機車約 15 分鐘
```

### 8-3. 修改靜態資訊（交通、房間說明等）

各類靜態說明文字位於：

| 想改的內容 | 修改的檔案 |
|------------|------------|
| 船班、租車、機車、自行車說明 | `handlers/transport_query.py` → `_TRANSPORT_INFO` 字典 |
| 空房查詢主選單文字 | `flex/room_menu.py` |
| 交通選單卡片文字 | `flex/theme_transport.py` |

修改後存檔，推送 GitHub，Render 自動重新部署（約 2–3 分鐘）。

### 8-4. 如何推送到 GitHub（更新上線）

如果負責維護的人員有安裝 Git：

```bash
git add .
git commit -m "更新行程資料"
git push origin main
```

Render 連接 GitHub 後，每次 push 都會**自動觸發重新部署**，無需手動操作。

## 9. GitHub 程式碼管理（Clone / Pull / Push）

本章說明如何在自己的電腦取得程式碼、同步更新，以及上傳修改。

### 9-1. 安裝必要工具

**安裝 Git（版本控制工具）**

1. 前往：https://git-scm.com/download/win
2. 下載 Windows 版安裝程式，一路點「Next」完成安裝
3. 安裝完成後，在桌面空白處按右鍵 → 點「**Git Bash Here**」，看到黑色視窗即成功

**安裝 VS Code（建議用來編輯程式碼）**

1. 前往：https://code.visualstudio.com/
2. 下載安裝，安裝時勾選「**Add to PATH**」選項

### 9-2. 設定 Git 身份（第一次使用才需要）

開啟 Git Bash，輸入以下兩行（將引號內換成自己的名字和 Email）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的Email@gmail.com"
```

### 9-3. Clone（第一次下載專案到本機）

「Clone」的意思是把 GitHub 上的專案完整複製到自己電腦。

**步驟：**

1. 前往 GitHub 專案頁面
2. 點綠色「**Code**」按鈕 → 複製 HTTPS 網址
   （格式如：`https://github.com/帳號/penghu-linebot.git`）
3. 在電腦選一個要放專案的資料夾，例如桌面，按右鍵 →「Git Bash Here」
4. 輸入以下指令（將網址換成剛才複製的）：

```bash
git clone https://github.com/帳號/penghu-linebot.git
```

5. 等待下載完成，資料夾內會出現整個專案
6. 進入專案資料夾：

```bash
cd penghu-linebot
```

> 之後每次要操作，都需先用 Git Bash 進入這個資料夾。

### 9-4. Pull（同步最新版本到本機）

當有人更新了 GitHub 上的程式碼（例如工程師修了 Bug），用 Pull 把最新版本下載到自己電腦。

**步驟：**

1. 在專案資料夾內開啟 Git Bash
2. 輸入：

```bash
git pull origin main
```

3. 看到「Already up to date.」表示本機已是最新版；有變動則會自動下載更新

> 建議每次修改資料前，先執行一次 Pull，確保是最新版本。

### 9-5. Push（上傳修改到 GitHub）

修改了本機的資料（例如新增行程、更新說明文字）後，用 Push 上傳到 GitHub，Render 會自動重新部署。

**完整步驟：**

```bash
# 第一步：查看哪些檔案有修改（可省略，但建議養成習慣）
git status

# 第二步：將所有修改加入準備區
git add .

# 第三步：寫一段說明（commit message，說明你改了什麼）
git commit -m "新增行程資料：澎湖藍洞半日遊"

# 第四步：上傳到 GitHub
git push origin main
```

4. 推送成功後，前往 Render Dashboard，會看到自動開始部署（約 2–3 分鐘完成）

### 9-6. 常用指令速查表

| 指令 | 說明 |
|------|------|
| `git status` | 查看目前有哪些檔案被修改 |
| `git pull origin main` | 從 GitHub 下載最新版本 |
| `git add .` | 將所有修改加入準備上傳的清單 |
| `git add 檔名` | 只加入特定檔案 |
| `git commit -m "說明文字"` | 確認這次修改，並附上說明 |
| `git push origin main` | 上傳到 GitHub |
| `git log --oneline` | 查看歷史修改紀錄 |

### 9-7. 注意事項

- **`.env` 檔案絕對不可 Push 到 GitHub**，此檔案包含所有密碼和金鑰，系統已設定自動忽略
- 每次 Push 前建議先 Pull，避免版本衝突
- 如果出現衝突（conflict）錯誤，請聯繫原開發人員協助處理
- GitHub 帳號需要有此專案的「**Collaborator**」權限才能 Push，請請原開發人員設定

## 10. 多人同時使用注意事項

### 目前限制

系統現階段部署在 Render **免費單一伺服器**上，能夠應對一般日常流量（數十人同時使用）。但在以下情境可能出現延遲：

| 情境 | 影響 | 說明 |
|------|------|------|
| 多人同時使用 AI 行程規劃 | 等待時間延長 | Mistral API 免費版有請求限制 |
| 大量航班查詢 | 爬蟲排隊 | 爬蟲需時 10–30 秒 |
| 系統剛喚醒 | 首次回覆慢 | 免費方案有休眠機制 |

### 後續優化方向（交由工程師處理）

1. **升級 Render 付費方案** – 消除休眠問題，提升並行處理能力
2. **升級 Mistral 付費 API** – 移除請求限制，支援更多同時查詢
3. **加入 Redis 快取** – 快取航班查詢結果，相同路線不重複爬蟲
4. **使用正式資料庫（PostgreSQL）** – 取代現有 JSON 檔案儲存方式，支援更大量用戶
5. **水平擴展（多台伺服器）** – 系統負載過高時自動增加伺服器數量

## 11. 常見問題排解

### Bot 沒有回覆

1. 確認 Render 服務狀態（https://render.com/ → Dashboard）是否為「Live」
2. 確認 LINE Webhook URL 設定正確，且「Use webhook」已開啟
3. 查看 Render Logs（服務頁面 → Logs）是否有錯誤訊息

### 收不到詢價 Email

1. 確認 Gmail 的「應用程式密碼」是否正確填入 `.env` / Render 環境變數
2. 確認 `NOTIFY_EMAIL` 是否填入正確收件信箱
3. 檢查垃圾郵件資料夾

### 航班查詢沒有結果

1. 立榮航空有驗證碼機制，辨識率非 100%，部分查詢可能失敗，再試一次通常可成功
2. 確認查詢的城市與日期有實際班次（如澎湖→台中目前無直航）

### 真人客服模式用戶卡住

1. 前往 LINE OA Manager 查看該用戶對話
2. 直接回覆或請用戶輸入「結束客服」即可恢復 Bot 模式

### 需要重新啟動系統

在 Render Dashboard 點選你的服務 → 右上角「**Manual Deploy**」→「**Deploy latest commit**」即可重新啟動。
